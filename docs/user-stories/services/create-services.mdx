# Create service instances and subscriptions

This guide shows how to create service entitlements, instances, and subscriptions using the BTP provider. As examples, we'll use the SAP Message Queuing service and [SAP Business Application Studio](https://www.sap.com/germany/products/technology-platform/business-application-studio.html).

## 🚧 Prerequisites

- You've created a running control plane.
- You've [configured the BTP provider](/docs/crossplane-provider-btp/docs/user-stories/setup/configure-provider-btp).
- You've [created and set up a subaccount managed in code](/docs/crossplane-provider-btp/docs/user-stories/account/subaccount).
- You're connected to your control plane.

## Service instance

With the `Entitlement` and `ServiceInstance` custom resources, you can orchestrate [entitlements](https://help.sap.com/docs/btp/sap-business-technology-platform/entitlements-and-quotas?locale=en-US&version=LATEST) and service instances on SAP BTP. The following steps show how to entitle the Message Queuing service to your managed subaccount and then create a Message Queuing service instance. 

### Create an `Entitlement`

To entitle the Message Queuing service to your subaccount, apply the `Entitlement` custom resource as described below.

1. Save the following YAML content to `entitlement-msq.yaml`.

    ```yaml title="entitlement-msq.yaml"
    apiVersion: account.btp.sap.crossplane.io/v1alpha1
    kind: Entitlement
    metadata:
      name: msq-entitlement
    spec:
      forProvider:
        serviceName: message-queuing
        servicePlanName: standard
        amount: 1
        subaccountRef:
          name: my-subaccount
      providerConfigRef:
        name: account-provider-config
    ```

:::note Want to create an HANA Cloud instance?
Don't forget to set the `spec.forProvider.servicePlanUniqueIdentifier` field. Use `servicePlanUniqueIdentifier: hana-cloud-hana-sap_eu-de-1` for the SAP Europe Data Center or `servicePlanUniqueIdentifier: hana-cloud-hana` for other supported data centers.

Check out our [crossplane-provider-hana](https://github.com/SAP/crossplane-provider-hana).
:::

2. Apply `entitlement-msq.yaml` to your control plane:

    ```sh title="Run in Terminal"
    kubectl create -f entitlement-msq.yaml
    ```

### Create a `ServiceInstance`

Once the Message Queuing service is entitled, you can create your first managed service instance. The steps below show how to create a Message Queuing service instance using our `ServiceInstance` custom resource.

1. Save the following YAML content to `instance-msq.yaml`.

    <Tabs groupId="service-instance">
      <TabItem value="yaml" label="YAML params" default>
        ```yaml title="instance-msq.yaml"
        apiVersion: account.btp.sap.crossplane.io/v1alpha1
        kind: ServiceInstance
        metadata:
          name: message-instance
        spec:
          forProvider:
            name: message-instance
            serviceManagerRef:
              name: my-subaccount-service-manager
            subaccountRef:
              name: my-subaccount
            offeringName: message-queuing
            planName: standard
            parameters:
              sourceIPs: 0.0.0.0/0
              resources:
                units: "100"
          providerConfigRef:
            name: account-provider-config # Please reference your providerConfig here!
        ```
      </TabItem>
      <TabItem value="json" label="JSON params">

        ```yaml title="instance-msq.json"
        apiVersion: account.btp.sap.crossplane.io/v1alpha1
        kind: ServiceInstance
        metadata:
          name: message-instance
        spec:
          forProvider:
        name: message-instance
        serviceManagerRef:
          name: my-subaccount-service-manager
        subaccountRef:
          name: my-subaccount
        offeringName: message-queuing
        planName: standard
        parameters: |
          {
            "sourceIPs": "0.0.0.0/0",
            "resources": {
              "units": "100"
            }
          }
        providerConfigRef:
          name: account-provider-config # Please reference your providerConfig here!
        ```

      </TabItem>
    </Tabs>

2. Apply `instance-msq.yaml` to your control plane:

    ```sh title="Run in Terminal"
    kubectl create -f instance-msq.yaml
    ```

That's it! You now have a managed instance of the Message Queuing service. Your Message Queuing service instance will be visible in the SAP BTP cockpit too.

## Subscription

With the BTP provider, you can also subscribe to SAP BTP services. The following example guides you through creating a managed subscription to SAP Business Application Studio.

### Create an `Entitlement`

Before subscribing to Business Application Studio, you must entitle the service to your subaccount.

1. Save the following YAML content as `entitlement-bas.yaml`.

    ```yaml title="entitlement-bas.yaml"
    apiVersion: account.btp.sap.crossplane.io/v1alpha1
    kind: Entitlement
    metadata:
      name: bas-entitlement
    spec:
      forProvider:
        serviceName: sapappstudio
        servicePlanName: standard-edition
        enable: true
        subaccountRef:
          name: my-subaccount
      providerConfigRef:
        name: account-provider-config # Please reference your providerConfig here!  
    ```

2. Apply `entitlement-bas.yaml` to your control plane:

    ```sh title="Run in Terminal"
    kubectl create -f entitlement-bas.yaml
    ```

### Create a `Subscription`

Once the entitlement is in place, you can subscribe to Business Application Studio by using the `Subscription` custom resource.

1. Save the following YAML content as `subscription-sapappstudio.yaml`.

    ```yaml
    apiVersion: account.btp.sap.crossplane.io/v1alpha1
    kind: Subscription
    metadata:
      name: subscription-sapappstudio
    spec:
      forProvider:
        appName: sapappstudio
        planName: standard-edition
      cloudManagementRef:
        name: my-subaccount-cis
    ```


2. Apply `subscription-sapappstudio.yaml` to your control plane:

    ```sh title="Run in Terminal"
    kubectl create -f subscription-sapappstudio.yaml
    ```

## ⁉ FAQs

### What entitlements are available in my subaccount?
To list available services and plans in your subaccount, run the following commands.

```sh title="Run in Terminal"
#!/bin/sh
SUBACCOUNTID=$1

btp --format=json list accounts/entitlement --filter-by-subaccount $SUBACCOUNTID | jq "[.entitledServices[] | {service: .name, service_plans: (.servicePlans | map( { (.name|tostring): {quota: .amount,unlimited:.unlimited,beta:.beta} } ) | add)}]"
```
